# waiver-stamp v0 Implementation Plan

> **For agentic workers:** this plan is executed **in-session, autonomously** (not handed
> to a zero-context engineer). Each task is TDD: write the failing test → run it red →
> implement minimal code → run it green → commit. Final code is written test-first during
> execution; this document fixes the **file structure, build order, interfaces, and the
> test list per task** — the decomposition that makes the build safe to run unattended.

**Goal:** Implement the waiver-stamp v0 engine + commit-embedded waivers + per-commit PR
verification + stdio MCP server + the `refactor-with-waiver` skill + a token-economy
benchmark + a problem-leading README — everything in `docs/spec.md` §§1–21 except the
carved-out remainder (GitHub push, CI/CD, npm release, plugin-install testing).

**Architecture:** A single TypeScript library (`src/`) wrapping **ts-morph**. The CLI
(`commander`) and the MCP server (`@modelcontextprotocol/sdk`) are thin adapters over the
same library functions — one engine, no second trusted path (spec §4, §18.1). Git
interaction is via `git` subprocess into throwaway worktrees. The benchmark drives the
installed `claude` CLI headless.

**Tech Stack:** TypeScript (ESM, NodeNext), ts-morph 24, zod 3.25, commander 12,
`@modelcontextprotocol/sdk`, vitest, biome, semantic-release (already wired).

## Global Constraints (verbatim from spec / repo)

- Node ≥ 20; ESM (`"type": "module"`); pnpm 9.12.0.
- Schema single source of truth: `src/schema.ts` (Zod) → types via `z.infer` → JSON Schema
  generated by `pnpm gen:schema`, kept honest by the drift-guard test (spec §4).
- Waiver header pins `schema: "waiver-stamp/v0"` and `tool: "waiver-stamp@<x.y.z>"`; stamp
  refuses on tool mismatch (spec §5, §17.1).
- Emit through the **repo's** TypeScript + tsconfig (ts-morph `getEmitOutput`), compare
  modulo whitespace (spec §7).
- Selectors are symbol-resolved TSDoc declaration references, never `line:col` (spec §5.2).
- Exit codes: `0` ok/stamped/approve, `1` stamp-or-claim failure / REQUEST_CHANGES, `2`
  malformed/header-mismatch, `3` internal (spec §10, §17.2).
- Fail-closed everywhere; only APPROVE removes review; downside-bounded (spec §1, §17.2).
- v0 reproductive ops are app-internal, single-project, non-lib (spec §8).
- Commit author/format follows the repo's existing conventional-commit + husky setup.

## PR Plan (bookending verdict)

This is a single-session build, so tasks commit to the current branch directly; the
push/PR-split is part of the carved-out remainder.

## File Structure

**Engine (new — `src/engine/`):**
- `project.ts` — load a ts-morph `Project` from a checkout dir + its tsconfig; resolve a
  `Selector`/`NodeLocator` to a ts-morph `Node` (spec §5.2).
- `ops/rename.ts` — `rename` op (native `node.rename`).
- `ops/extract-function.ts` — `extract-function` via language-service "Extract Symbol".
- `ops/move-to-new-file.ts` — `move-to-new-file` via LS "Move to a new file".
- `ops/index.ts` — op dispatch table keyed by `op`.
- `fold.ts` — apply transform ops in order over a working copy.
- `emit-compare.ts` — emit changed files of `O` vs head through the program, compare
  modulo whitespace; return per-file equality (spec §7).
- `guards.ts` — static guards: single-project, public-API, dynamic-reference scan,
  emit-divergence (spec §8).

**Git (new):**
- `src/git.ts` — `worktreeAt(ref)`, `commitsInRange(base, head)`, `commitMessage(sha)`
  (`%B`), `parents(sha)`, `changedFiles(base, head)`.

**Commit-embedded waivers (new):**
- `src/commit-waiver.ts` — `extractWaiverBlock(message)` (parse, §17.1 algorithm),
  `embedWaiver(subject, waiver)` (produce a commit message with the fenced block).

**Verification (new):**
- `src/verify.ts` — walk `base..head`, classify each commit, aggregate verdict (§17.2).
- `src/commit.ts` — `waiver commit` command logic (§17.4).

**MCP (new):**
- `src/mcp.ts` — stdio MCP server exposing `waiver_check/apply/stamp/verify` (§18.1).

**Existing (extend):**
- `src/load.ts` — add `loadWaiverFromObject`.
- `src/report.ts` — add `VerifyReport`, `PerCommitResult`, `CommitClass`.
- `src/apply.ts` — implement over `engine/fold`.
- `src/stamp.ts` — implement over `engine/{fold,emit-compare,guards}` + git.
- `src/cli.ts` — add `verify`, `commit`, `mcp` subcommands.
- `src/schema.ts` — forbid ` ``` ` in the `tool`/string fields (§17.1 robustness).
- `src/index.ts` — export the new public surface.

**Plugin / skill / bench / docs (new or replaced):**
- `plugin/skills/refactor-with-waiver/SKILL.md` — new skill (§18.2).
- `plugin/.claude-plugin/plugin.json` — register the MCP server; drop generate-waiver.
- `bench/` — `tasks/`, `fixture/`, `mcp.json`, `rubric.md`, `results.json`, `results.md`.
- `scripts/bench.ts` — the benchmark driver (`pnpm bench`).
- `README.md` — rewrite per §20.

## PR 2 — Tasks

### Task 1 — Prep seam: `loadWaiverFromObject` + verify report types

**Files:** Modify `src/load.ts`, `src/report.ts`, `src/types.ts`; Test
`src/load.test.ts`, `src/report.test.ts`.

**Interfaces produced:**
- `loadWaiverFromObject(obj: unknown): Waiver` — schema-validates an already-parsed object
  (throws `MalformedWaiverError` on failure). `loadWaiver(path)` is refactored to read the
  file, `JSON.parse`, and delegate here.
- `type CommitClass = 'stamped' | 'invalid' | 'unwaivered' | 'skipped'`
- `interface PerCommitResult { sha: string; subject: string; class: CommitClass; reasons: string[]; perOpFindings: OpFinding[]; uncoveredFiles: string[] }`
- `type Verdict = 'APPROVE' | 'COMMENT' | 'REQUEST_CHANGES' | 'ABSTAIN'`
- `interface VerifyReport { verdict: Verdict; commits: PerCommitResult[] }`

**Tests:** object that matches schema → returns typed waiver; object with bad `op` →
throws `MalformedWaiverError`; `loadWaiver` still works via the new delegate.

### Task 2 — Engine: load a ts-morph Project + resolve a top-level `Selector`

**Files:** Create `src/engine/project.ts`, `src/engine/project.test.ts`; a tiny TS fixture
under `src/engine/__fixtures__/`.

**Interfaces produced:**
- `loadProject(cwd: string): Project` — finds the nearest `tsconfig.json`, returns a
  ts-morph `Project` (in-memory FS off; reads real files).
- `resolveSelector(project, sel: Selector): Node` — resolves `{file, symbol}` where
  `symbol` is a top-level name; throws `SelectorResolutionError` if 0 or >1 match.

**Tests:** resolve a top-level function by name; resolve a class member `Class.method`;
unknown symbol → throws; ambiguous → throws. (Disambiguation grammar `:static`/`:instance`/
overload-index is added in Task 2b only if a test needs it — YAGNI; start with the common
top-level + member case.)

### Task 3 — Engine op: `rename`

**Files:** Create `src/engine/ops/rename.ts`, test; reuse fixtures.

**Interfaces produced:** `applyRename(project, op: RenameOp): void` — resolves the target,
calls `node.rename(op.to)`, throws on collision.

**Tests:** rename a function updates its declaration + all in-program references; rename
into an existing name in scope → throws; rename a class method updates call sites.

### Task 4 — Engine: `fold` transform ops + implement `apply`

**Files:** Create `src/engine/fold.ts`, `src/engine/ops/index.ts`, tests; rewrite
`src/apply.ts`; update `src/apply.test.ts`.

**Interfaces produced:**
- `foldOps(project, ops: TransformOp[]): void` — dispatches each transform op in order via
  the op table (rename now; extract/move/bump added in later tasks register here).
- `apply(path, {cwd}): Promise<{files: string[]}>` — load waiver, `loadProject(cwd)`,
  `foldOps`, `project.save()`, return changed file paths. No longer throws NotImplemented.

**Tests:** apply a one-rename waiver to a fixture dir → file content shows the rename; apply
returns the changed file list; a waiver with an exclusion-only op changes nothing.

### Task 5 — Git helpers + throwaway worktrees

**Files:** Create `src/git.ts`, `src/git.test.ts` (uses a temp git repo built in the test).

**Interfaces produced:** `commitsInRange`, `parents`, `commitMessage(sha)` (uses `%B`),
`changedFiles(base, head)`, `worktreeAt(repo, ref): Promise<{dir, cleanup}>`.

**Tests:** build a temp repo with 3 commits; `commitsInRange` returns them in order;
`commitMessage` returns the full body incl. fenced block; `worktreeAt` checks out a ref to
a temp dir and `cleanup` removes it; merge commit reports 2 parents.

### Task 6 — Engine: `emit-compare` + implement `stamp` (single waiver file)

**Files:** Create `src/engine/emit-compare.ts`, test; rewrite `src/stamp.ts`; update
`src/stamp.test.ts`.

**Interfaces produced:**
- `emitOf(project, file): string` — `getEmitOutput` for one file, whitespace-normalized.
- `compareEmit(oProject, headProject, files): {file, equal}[]`.
- `stamp(waiver, {base, head, cwd}): Promise<StampReport>` — worktree base + head, fold ops
  over base→`O`, compute the changed-file set (base vs head), require every changed
  production file be either reproduced (emit `O` == emit head) or covered by an exclusion
  op; else FAIL. Returns the §10 report.

**Tests (the paramount-feature proof):** a fixture repo where head = `apply`(rename waiver)
over base → `stamp` PASSES; head with an extra un-accounted hand edit → FAILS with the
uncovered file listed; head missing one renamed reference → FAILS (emit mismatch).

### Task 7 — Engine op: `extract-function`

**Files:** Create `src/engine/ops/extract-function.ts`, test; register in op table.

**Interfaces produced:** `applyExtractFunction(project, op)` — LS "Extract Symbol" to
narrowest enclosing scope via `getApplicableRefactors`, apply edits, rename generated
`newFunction` → `op.name` (determinism, spec §6.1, §9).

**Tests:** extract a statement span into a named function; the call site invokes it; 
emit-compare against a hand-equivalent head passes; ambiguous action → fails.

### Task 8 — Engine op: `move-to-new-file`

**Files:** Create `src/engine/ops/move-to-new-file.ts`, test; register.

**Interfaces produced:** `applyMoveToNewFile(project, op)` — LS "Move to a new file" for the
named symbols, rewires imports/exports.

**Tests:** move two decls to a new file; importers are rewired; emit-compare passes.

### Task 9 — Static guards

**Files:** Create `src/engine/guards.ts`, test; call from `stamp` before fold.

**Interfaces produced:** `runGuards(project, ops): GuardFinding[]` — single-project
(reference outside loaded program → FAIL), public-API (`libs/*-sdk|*-api-contract`
`index.ts` export → FAIL), dynamic-reference scan (`obj["name"]`, string-keyed refs,
non-TS sibling refs → FAIL), emit-divergence (changed file touches decorator-metadata /
`const enum` / class-fields / `namespace`/`import =` → FAIL). Stamp FAILs if any.

**Tests:** one positive + one negative per guard.

### Task 10 — `commit-waiver`: parse + embed (spec §17.1)

**Files:** Create `src/commit-waiver.ts`, `src/commit-waiver.test.ts`.

**Interfaces produced:**
- `extractWaiverBlock(message): {kind:'none'} | {kind:'one', waiver:Waiver} | {kind:'invalid', reason:string}` — scan **all** ` ```json ` blocks, select those with `schema === "waiver-stamp/v0"` exactly; 0→none, 1→validate (schema + 64KiB + tool-version-at-caller), ≥2→invalid; tolerant of CRLF/trailing ws.
- `embedWaiver(subject, waiver): string` — subject + blank line + fenced json block.

**Tests:** message with one valid block → one; decoy non-waiver json + one waiver → one;
two waiver blocks → invalid; block with `schema:"waiver-stamp/v9"` → none (not a v0 waiver
→ unwaivered) ⟹ documented; malformed json with schema key → invalid; >64KiB → invalid;
round-trip `extractWaiverBlock(embedWaiver(s,w)).waiver` deep-equals `w`.

### Task 11 — `verify`: per-commit classification + aggregation (spec §17.2)

**Files:** Create `src/verify.ts`, `src/verify.test.ts`.

**Interfaces produced:** `verify({base, head, cwd}): Promise<VerifyReport>` — for each
commit in range: skip merge/out-of-range (class `skipped`); else `extractWaiverBlock` →
unwaivered / invalid / (validate tool-version → invalid `tool-mismatch`) / `stamp` with
`base=C^1, head=C` → stamped|invalid. Aggregate by precedence
REQUEST_CHANGES>COMMENT>APPROVE>ABSTAIN.

**Tests (the verdict matrix):** all-stamped → APPROVE; one stamped + one unwaivered →
COMMENT; one invalid (anything else) → REQUEST_CHANGES; zero waivers → ABSTAIN; a merge
commit in range is skipped, not failed; tool-version mismatch → invalid → REQUEST_CHANGES.

### Task 12 — `waiver commit` command (spec §17.4)

**Files:** Create `src/commit.ts`, test.

**Interfaces produced:** `commitWaiver(path, {subject, cwd}): Promise<{sha}>` — refuse on
dirty tree; `check`; `apply`; `git add` changed files; `git commit -m embedWaiver(...)`.

**Tests:** clean tree + valid rename waiver → a commit whose message round-trips the waiver
and whose tree has the rename; dirty tree → throws; failing `check` → throws, no commit.

### Task 13 — Wire CLI: `verify`, `commit`, `mcp`; exit codes

**Files:** Modify `src/cli.ts`, `src/index.ts`; add `src/cli.test.ts` cases.

**Tests:** `verify` exit 0 on APPROVE, 1 on REQUEST_CHANGES; `--json` emits the report;
`commit` writes a commit; `mcp` boots the server and responds to `tools/list` (smoke).

### Task 14 — MCP server (spec §18.1)

**Files:** Create `src/mcp.ts`, `src/mcp.test.ts`; add `@modelcontextprotocol/sdk` dep.

**Interfaces produced:** `createServer(): McpServer` exposing
`waiver_check/apply/stamp/verify` with inline-`waiver` + `cwd?` inputs, returning the JSON
reports. `waiver mcp` starts it on stdio.

**Tests:** in-process client calls `waiver_check` with a valid waiver → ok result; with a
bad waiver → error result; `tools/list` returns the four tools.
